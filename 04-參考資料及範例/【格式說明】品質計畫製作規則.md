## 零、依契約金額決定品質計畫最低章節要求（公共工程施工品質管理制度）

> 依據：「公共工程施工品質管理標準作業要點」整體品質計畫審查重點表
> 參考文件：`整體品質計畫章節及重點.docx`（已存放於本資料夾）

### 📋 契約金額分級對照表

| 契約金額區間 | 最低應包含章節 |
|------------|-------------|
| **150萬以上，未達1,000萬** | 管理權責分工、品質管理標準、材料及施工檢驗程序、自主檢查表 |
| **1,000萬以上，未達5,000萬** | 計畫範圍、管理權責分工、品質管理標準、材料及施工檢驗程序、自主檢查表、文件紀錄管理系統 |
| **5,000萬以上** | 計畫範圍、管理權責分工、**施工要領**、品質管理標準、材料及施工檢驗程序、設備功能運轉檢測程序及標準、自主檢查表、**不合格品之管制**、**矯正與預防措施**、**內部品質稽核**、文件紀錄管理系統（共11項） |

> ⚠️ **備註**：
> - 工程具運轉類機電設備且金額達150萬以上者，均須增訂「設備功能運轉測試等抽驗程序及標準」
> - 金質獎工程通常屬5,000萬以上等級，故採完整11章（或整合為8/9章制）
> - 分項品質計畫應包括：品質管理標準、材料及施工檢驗程序、施工要領（含要領表格）、自主檢查表

### 🔑 自主檢查表必要欄位（官方審查要求）

自主檢查表每份必須包含以下欄位（缺一不可）：
1. 工程名稱
2. 分項工程名稱
3. 協力廠商
4. 檢查位置
5. 檢查日期
6. 檢查時機（施工前/施工中/施工完成）
7. 檢查結果符號說明（○合格 ╳不合格 ／不適用）
8. 檢查項目（含施工前/施工中/施工完成三階段）
9. 檢查標準（依設計圖說及規範，以數字量化）
10. 實際檢查情形（加註量化數據）
11. 檢查結果（○╳／）
12. 缺失矯正處理
13. 缺失複查結果（□已完成改善 □未完成改善）
14. 備註
15. 檢查人員簽名（現場工程師＋工地負責人，品管人員不列入）

> ⚠️ 品管人員**不得**列入自主檢查表的簽名欄位

## 二、各章標準格式

### 第一章 計畫範圍

**前言段落格式：**
> ○○營造有限公司（以下簡稱本公司）承攬【工程名稱】，為達成契約所規定之工程品質目標，依據「公共工程施工品質管理制度及作業要點」建立施工品質管制系統，設立品質管理組織，訂定整體品質計畫，據以推動施實。

**依據（一）節：** 條列式，包含工程契約、技師法、建築法、營造業法、公共工程專業技師簽證規則、職業安全衛生法等。

**工程概要（二）節標準欄位：**
- 工程名稱、主辦機關、設計單位、監造單位、承攬廠商
- 工程地點、契約工期（日曆天）、契約金額
- 工程內容（A區、B-1區…分區說明）
- **表 一-1 工程主要施工項目及數量**（欄位：項次｜項目及說明｜單位｜數量）

**名詞定義（四）節：** 條列28項以上，包含本工程、主辦機關、設計廠商、監造單位、承攬廠商、工地主任、專任工程人員、品管人員、現場工程師、程序、檢驗、自主檢驗、確認、品質、施工缺失、內部品質稽核、矯正措施、**檢驗停留點**、**安衛查驗點**、管理標準等。

### 第二章 管理責任及權責分工

**品管組織架構圖（圖二-1）格式：**
```
承攬廠商管理階層（主任技師）
        ↓
    工地負責人（工地主任）─── 職業安全衛生管理人員
        ↓
品管人員  現場工程師  行政人員
        ↓
    協力廠商
```

**表 二-1 工作職掌表欄位：** 職稱｜工作職掌及依據

職稱包含：管理階層、專任工程人員（主任技師）、工地負責人（工地主任）、品管人員、職業安全衛生管理人員、現場工程師、行政人員

**人員輪值：** 依勞基法施行細則第23條之1規定，趕工時輪派留守，請假時覓妥合格人員代理。

**管理審查：** 每季審查品質管理系統，內容包括前次審查執行情形、內部品質稽核、矯正/預防措施、外部驗證及工程評鑑結果。

**表 二-2：** 公共工程施工中營造業專任工程人員督察紀錄表（編號EP-）

### 第七章 自主檢查表

**每份自主檢查表固定格式（編號EW-XX）：**
```
工程名稱：【工程名稱全名】
分項工程名稱：＿＿＿＿　　協力廠商：＿＿＿＿
檢查位置：＿＿＿＿　　　　檢查日期：＿年＿月＿日
施工流程：□施工前檢查　□施工中檢查　□施工完成檢查
檢查結果：○檢查合格　╳有缺失需改正　／無此檢查項目
```

**表格欄位：**
| 檢查項目 | 設計圖說、規範之檢查標準（定量定性） | 實際檢查情形（敘述檢查值） | 檢查結果 |

**底部固定格式：**
```
缺失複查結果：
□已完成改善
□未完成改善，填至「材料/設備檢（試）驗管制總表」第○項進行追蹤改善
複查日期：＿年＿月＿日
複查人員職稱：＿＿＿　簽名：＿＿＿

備註：
1.檢測標準及實際檢測情形應具體明確或量化尺寸。
2.檢測結果合格者註明「○」，不合格者註明「╳」，如無需檢測之項目則打「／」。
3.嚴重缺失、缺失複查未能及時完成改善，應填具「不合格品管制總表」進行追蹤改善，本表單可先行存檔。
4.本表由工地現場施工人員實地檢測後覈實記載簽認

工地主任(工地負責人)簽名：＿＿＿　　現場施工人員（檢查人員）簽名：＿＿＿
```

**核心函式：**
```python
def _char_font(ch, cjk_f, ascii_f):
    """根據 cmap 選擇正確字體"""
    return cjk_f if ord(ch) in _cmap_cjk else (ascii_f if ord(ch) in _cmap_ascii else cjk_f)

def dtext(draw, xy, text, font, fill):
    """逐字選字體渲染，解決方框問題"""
    x, y = xy
    for ch in text:
        f = _char_font(ch, font, get_ascii_font(font.size))
        draw.text((x, y), ch, font=f, fill=fill)
        x += f.getbbox(ch)[2] - f.getbbox(ch)[0]
```

### python-docx 已知 Bug 修正

**Bug 1：w:shd 元素位置錯誤（儲存格背景色）**
- 症狀：Word 開啟時顯示「已修復」警告
- 原因：python-docx 將 `w:shd` 放在 `tcPr` 末尾，但 Word 要求在 `noWrap/vAlign/hideMark` 之前
- 修正：`set_cell_bg()` 函式中用 `tcPr.insertBefore(shd, first_illegal)` 強制插入正確位置

**Bug 2：w:zoom 屬性錯誤（文件縮放）**
- 症狀：文件開啟時縮放比例異常
- 原因：python-docx 生成 `<w:zoom w:val="bestFit"/>` 而非有效屬性
- 修正：存檔後以 zipfile 開啟，用 `re.sub` 替換 settings.xml 中的 zoom 設定

```python
# 正確修正方式（不使用 os.remove，改用 os.replace）
import zipfile, re, os
tmp = out_path + ".tmp"
with zipfile.ZipFile(out_path,'r') as zi, zipfile.ZipFile(tmp,'w',zipfile.ZIP_DEFLATED) as zo:
    for item in zi.infolist():
        data = zi.read(item.filename)
        if item.filename == 'word/settings.xml':
            data = re.sub(b'<w:zoom[^/]*/>', b'<w:zoom w:percent="100"/>', data)
        zo.writestr(item, data)
os.replace(tmp, out_path)   # 用 os.replace 而非 os.remove + rename（避免 PermissionError）
```

### 腳本內 fc' 符號的 Python 語法陷阱

混凝土設計強度 `fc'=280 kgf/cm²` 中含有單引號，在 Python 單引號字串內會造成 SyntaxError：
```python
# ❌ 錯誤（導致 SyntaxError: invalid decimal literal）
('混凝土澆置（fc'=280kgf/cm²）', ...)

# ✅ 正確（跳脫單引號）
('混凝土澆置（fc\'=280kgf/cm²）', ...)

# ✅ 或用雙引號包覆整個字串
("混凝土澆置（fc'=280kgf/cm²）", ...)
```
