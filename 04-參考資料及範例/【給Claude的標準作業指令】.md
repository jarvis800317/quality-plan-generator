## 🖋️ 字型與版面規格（【必填】每次依當次上傳監造計畫分析後填入）

> ⚠️ **品質計畫的封面、目錄、標題、內文及附件的字體、字型、大小等格式，
> 必須與當次上傳監造計畫的對應格式完全一致。**
> 每次新工程開始前，Claude 必須先分析監造計畫字型，再更新腳本設定區常數，
> 不得沿用前一個工程的字型設定。

### 字型分析步驟（每次新工程必執行）

1. 讀取 `01_監造計畫\` 內的監造計畫（PDF 或 DOCX）
2. 分析並記錄下表各項目（PDF 用 fontTools，DOCX 用 python-docx 讀取 styles）
3. 將結果填入腳本設定區（`DOCX_FONT_CJK`、`DOCX_FONT_LATIN`、`DOCX_SZ_*` 各常數）

## 🚨 四項強制要求（每次作業前必須確認）

> **⚡ 效率提示：日後新工程優先採用「腳本複用模式」（見本檔最末節），可節省 90% 以上 context 用量。**

### 強制要求零：禁止讀取原始 PDF / DOCX 參考檔案

資料夾內的大型 PDF 和 DOCX（如平鎮運動公園-整體品質計畫.pdf 49MB、頂福安居整體品質計畫.pdf 等）**禁止主動讀取**。
這些原始檔案每份讀取需消耗數萬 tokens，且資訊已萃取完畢。

**只讀以下輕量檔案（依序優先）：**
1. `金質獎品質計畫標準分析_*.md`（8～10KB，已萃取的精華）
2. `【給Claude的標準作業指令】.md`（本檔）
3. `【監造計畫與品質計畫對應關係】.md`
4. `QualityPlanMaker\01_監造計畫\本工程監造計畫`（本次工程的監造計畫，必要）

**例外：** 當被明確指示「製作 MD 摘要」時，才讀取特定原始 PDF，但僅讀取一次，不重複。

### 強制要求一：章節內容直接寫入 Python 腳本，不在 chat 完整展示

**正確做法（節省 context 的方式）：**
- 每章完成後，自動繼續下一章，無需等待使用者輸入；全部完成後在 chat 回覆：「✅ 全部章節已完成，執行腳本中」
- 章節的完整文字、表格、自主檢查表項目**全部直接寫入 Python 腳本**（python-docx 格式）
- 禁止在 chat 內逐字輸出整章內容（這是 context 的最大浪費來源）

**允許在 chat 簡短確認的內容（每章不超過 15 行）：**
- 本章主要施工工項清單
- 特殊品管數值（如混凝土坍度、壓實度等）
- 流程圖結構大綱
- 詢問工程特定資料（如廠商名稱、工期等）

### 強制要求二：每章應繪製的流程圖一律用 Python PIL 產生 PNG 格式

**⚠️ 重要：sandbox 環境中無法使用 cairosvg / inkscape / rsvg-convert，SVG 無法嵌入 DOCX。**
**必須使用 Python PIL 直接生成 PNG 影像，再透過 python-docx 嵌入文件。**

**PIL 流程圖技術規格（必須遵守）：**
```python
# 雙字體方案（缺一不可）
FONT_CJK   = "/usr/share/fonts/truetype/droid/DroidSansFallbackFull.ttf"  # 中文 + 全形符號
FONT_ASCII = "/usr/share/fonts/truetype/dejavu/DejaVuSans.ttf"             # ASCII + 數學符號

# 說明：DroidSansFallbackFull 只含中文字形，缺少 -、1、%、±、≦、→、℃ 等
# 若只用單一字體，這些符號會顯示為方框 □
# 必須逐字判斷使用哪個字體（_char_font 函式）
```

### 強制要求三：腳本後處理修正（python-docx 已知 Bug）

**每次生成 DOCX 後必須執行兩項 XML 修正：**

```python
# 修正一：w:shd 元素位置（儲存格背景色）
# python-docx 將 w:shd 放在錯誤位置，導致 Word 開啟報錯
# 正確位置：w:shd 必須在 noWrap / tcMar / vAlign / hideMark 之前
# → set_cell_bg() 函式中用 insertBefore 處理

# 修正二：w:zoom 屬性（文件縮放設定）
# python-docx 生成 <w:zoom w:val="bestFit"/>（錯誤）
# 必須改為 <w:zoom w:percent="100"/>
# → 儲存後用 zipfile 開啟 word/settings.xml，用 re.sub 替換
import zipfile, re
with zipfile.ZipFile(out_path,'r') as zi, zipfile.ZipFile(tmp,'w',zipfile.ZIP_DEFLATED) as zo:
    for item in zi.infolist():
        data = zi.read(item.filename)
        if item.filename == 'word/settings.xml':
            data = re.sub(b'<w:zoom[^/]*/>', b'<w:zoom w:percent="100"/>', data)
        zo.writestr(item, data)
os.replace(tmp, out_path)  # 用 os.replace 而非 os.remove（避免權限錯誤）
```
