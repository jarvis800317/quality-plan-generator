# 章節選擇流程（Step 4）

## 適用時機

每次產出品質計畫前，在「顯示工程概要確認表（Step 3）」之後、
「選擇腳本模式（Step 5）」之前，必須執行本章節選擇步驟。

---

## 工具定位說明（必讀）

| 工具 | 定位 | Claude 能否讀取 |
|------|------|----------------|
| **chapter_selector.jsx** | 主要決策工具。使用者在此輸入契約金額、勾選章節，再點「📋 複製配置」。 | ❌ 直接讀取（前端狀態不回傳）|
| **Claude 對話框（貼上文字）** | 傳遞決策的唯一管道。使用者將 JSX 產生的配置文字貼入對話框，Claude 即可解析。 | ✅ **能**（讀取貼上的文字） |

> ✅ 正確流程：**JSX 選擇 → 複製 → 貼到對話框 → Claude 解析**
> 不再需要 AskUserQuestion，主任技師在 JSX 介面完成所有決策後，只需一次貼上即可。

---

## 執行流程

### 1. 帶入契約金額 & 計算建議章節

從監造計畫已讀取的契約金額（**單位：元**），對照下表判定級距與必選章節：

| 級距（元） | 必選章節（鎖定） | 說明 |
|-----------|--------------|-----|
| 50,000,000 元以上（5,000萬以上） | 計畫範圍、管理責任及權責分工、施工要領、品質管理標準、材料及施工檢驗程序、自主檢查表、不合格品之管制、矯正與預防措施、內部品質稽核、文件紀錄管理系統（共10章） | 設備功能運轉測試為選配 |
| 10,000,000 元以上，未達 50,000,000 元（1,000萬～未達5,000萬） | 計畫範圍、管理責任及權責分工、品質管理標準、材料及施工檢驗程序、自主檢查表、文件紀錄管理系統（共6章） | 其餘可選配增訂 |
| 1,500,000 元以上，未達 10,000,000 元（150萬～未達1,000萬） | 管理責任及權責分工、品質管理標準、材料及施工檢驗程序、自主檢查表（共4章） | 其餘可選配增訂 |

---

### 2. 呈現視覺化章節選擇器（JSX）並等待貼上

#### 2-A. JSX 檔案說明

`chapter_selector.jsx` 已永久存放於工作區資料夾，**不需要每次重新產生**。
使用者在介面中：
1. 確認或修改契約金額（欄位預帶偵測值）
2. 勾選/取消章節（🔒 必選自動鎖定）
3. 點「📋 複製配置到 Claude 對話框」
4. 將複製的文字貼回 Claude 對話框

#### 2-B. 呈現連結格式

Claude 必須動態填入當前 SESSION_ID，格式如下：

```
📋 請開啟章節選擇器，確認契約金額與章節後，點「複製配置」貼回此對話框：
[開啟章節選擇器](computer:///sessions/{當前SESSION_ID}/mnt/QualityPlanMaker/chapter_selector.jsx)

操作步驟：
① 畫面中確認或修改契約金額（預帶值：{偵測金額} 元）
② 勾選要納入的章節（🔒 必選自動鎖定，➕ 選配可自由取捨）
③ 點「📋 複製配置到 Claude 對話框」
④ 將複製的文字貼回此對話框，Claude 即自動讀取並產出文件
```

> ⚠️ **禁止複製其他 session 的路徑**。SESSION_ID 取自當前 `OUT_PATH` 的
> `/sessions/xxxxx/` 段落，例如 `OUT_PATH` 為
> `/sessions/epic-keen-maxwell/mnt/QualityPlanMaker/...` 時，
> SESSION_ID = `epic-keen-maxwell`

#### 2-C. Claude 等待貼上

呈現連結後，Claude **靜待**使用者將配置文字貼回對話框，**不得自行假設配置或呼叫 AskUserQuestion**。

---

### 3. 解析使用者貼上的配置文字

使用者從 JSX 複製後貼至對話框的文字格式如下：

```
【章節配置確認 — 請將以下文字貼至 Claude 對話框】
契約金額：5,822,919 元（150萬～未達1,000萬，法規必選 4 章）
納入章節（共 6 章）：壹 計畫範圍、貳 管理責任及權責分工、肆 品質管理標準、伍 材料及施工檢驗程序、柒 自主檢查表、拾壹 文件紀錄管理系統
排除章節：參 施工要領、陸 設備功能運轉測試、捌 不合格品之管制、玖 矯正與預防措施、拾 內部品質稽核
CHAPTER_FLAGS=scope:T,mgmt:T,method:F,standard:T,inspection:T,equipment:F,selfcheck:T,nc:F,ca:F,audit:F,doc:T
```

#### Claude 解析規則

1. **讀取 `CHAPTER_FLAGS=` 一行**，將 `key:T` 解析為 `True`、`key:F` 解析為 `False`
2. **檢查必選章節**：若必選 key 對應值為 `F`，自動改為 `True` 並在 chat 提醒：「⚠️ 依法規，XX 章為必選，已自動補回。」
3. **在 chat 列出確認摘要**（見下方 §3.5），再進入 Step 4

#### 例外情況處理

- **貼上文字格式不完整**：Claude 可詢問「您的配置文字似乎不完整，請重新從章節選擇器複製後貼上，或直接告訴我想納入哪些章節。」
- **使用者直接用文字說明**（未使用 JSX）：Claude 依使用者說明解析章節清單，整理後列出確認摘要供確認。

---

### 3.5. 收到配置後的確認摘要（每次必執行）

解析完成後，Claude 必須在 chat 列出最終配置摘要，**不得直接進入 Step 4**：

```
【章節配置確認】
契約金額：{金額} 元（{級距文字}）

✅ 納入章節（共 N 章）：
  壹　計畫範圍 ／ 貳　管理責任及權責分工 ／ … （依序列出）

❌ 排除章節：
  參　施工要領 ／ 陸　設備功能運轉測試 ／ … （若無則填「無」）

⚠️ 自動補回章節（必選但原標記為 F）：
  … （若無則略去此行）
```

確認摘要列出後，**直接進入 Step 4 更新 CHAPTER_FLAGS**，無需再詢問使用者。
若使用者提出修改，請其在 JSX 重新選擇後再次貼上，或直接用文字告知調整內容。

---

### 4. 依回應更新 CHAPTER_FLAGS

使用者確認後，將以下常數更新至 Python 腳本的 `CHAPTER_FLAGS` 區塊：

```python
# ═══════════════════════════════════════════════════
# ⭐ 章節選擇（依使用者確認後填入 True/False）
# ═══════════════════════════════════════════════════
CHAPTER_FLAGS = {
    'scope':      True,   # 壹　計畫範圍
    'mgmt':       True,   # 貳　管理責任及權責分工
    'method':     True,   # 參　施工要領
    'standard':   True,   # 肆　品質管理標準
    'inspection': True,   # 伍　材料及施工檢驗程序
    'equipment':  True,   # 陸　設備功能運轉測試（選配）
    'selfcheck':  True,   # 柒　自主檢查表
    'nc':         True,   # 捌　不合格品之管制
    'ca':         True,   # 玖　矯正與預防措施
    'audit':      True,   # 拾　內部品質稽核
    'doc':        True,   # 拾壹　文件紀錄管理系統
}
```

腳本會根據 True/False 自動：
1. 決定是否產出該章節
2. 動態重新計算章序（壹、貳、參…不跳號）
3. 更新目錄頁碼對應

## 注意事項

1. **必選章節不得排除**：若使用者要求排除必選章節，Claude 應說明法規限制，並建議改為「精簡內容」而非「刪除章節」。
2. **設備功能運轉測試**：有景觀照明、噴灌、排水設備等運轉類機電設備時，強烈建議保留；若排除，應在文件中備註理由。
3. **全 11 章勾選設計**：主任技師在 `chapter_selector.jsx` 畫面中逐章勾選（🔒 必選自動鎖定，➕ 選配可自由取捨），點「複製配置」後貼回對話框，Claude 自動解析。**禁止使用 AskUserQuestion 代替 JSX 選擇流程。**
4. **金額單位**：全程使用「元」，禁止使用「萬元」。從監造計畫讀取的契約金額須完整帶入（如：35,000,000 元），視覺化選單欄位自動預填。
5. **法規提醒**：1,500,000 元以上，未達 10,000,000 元工程，貳管理責任及權責分工、肆品質管理標準、伍材料及施工檢驗程序、柒自主檢查表為**法定最低必選 4 章，不得刪除或合併省略**。
6. **JSX 連結路徑**：每次必須動態填入當前 session 名稱，**禁止複製其他 session 的路徑**。chapter_selector.jsx 已永久存放於工作區，不需重新產生，僅需更新連結中的 SESSION_ID。
